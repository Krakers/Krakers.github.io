What exactly is git?

git is a version control system created by Linus Torvalds:

What's a version control system anyway?

  • Version Control Systems (VCS for short) are programs that keep track of changes in our files
  • Because of them, multiple people can work on the same files, without colliding with one another
  • They don't have to be source code files, designers use git too!
Why do we need it?

We need git for following reasons:

  • Since we have a lot of files in our project, we need to make sure that everyone has exactly the same version
  • We need to ensure that multiple people can work on a single file (like in automated tests)
  • We need a system that handles all file changes for us, using email, FTP or pigeon mail is our of the question

Imagine a use case where multiple people are trying to work on a single novel

  • We might use something like Google Docs but editing the same document simultaneously is not convienent
  • We also could make them work on their own computers and then merge all created .doc files (somehow)
  • By using git, we can avoid those problems. Everyone can work on their own copy of the repository and easily collaborate with one another

First lets learn some definitions
What is a repository?
  • A repository is a directory where your projects can live.
  • Most git users shorten this to “repo”.
  • It can be local to a folder on your computer, or it can be a storage space on GitHub or another online host.
  • You can keep code files, text files, image files, you name it, inside a repository.
What is a commit?
  • git commit is the command that gives Git its power.
  • A commit is a "snapshot" of your respository at that point in time.
  • Those "snapshots" are extremely useful because you can restore your projects' state at any point in time
  • Each commit has a commit message and a unique hash code which lets git remember your projects' state at this point

Bear in mind that other people can and will read our commit messages

What is a branch?
  • In order to be able to cooperate with each other we need to be able to work indepedently from one another.
  • That's why we need branches.
  • They allow us to "branch off" of the main project and add our changes to the files without worrying about the rest.
  • After we finish working on our branch, it's time to "merge" that branch with the master branch

Okay, now what?
Now we need to talk about git states (Listen carefully, this is important)

In git, there are four states that a file can be in

  • Untracked file
  • Modified file
  • Staged file
  • Commmitted file
To check the current status of our files we use git status command We're going to see it in action in a minute

Untracked file

  • Untracked file means that git will not keep track of any changes in this file
  • All new files are untracked by default
  • In order to make git aware of those files we need to add them to repository using git add command
